## 二、参考素材生产流程

### 2.1 产品参考图生成（最关键环节）

**目标**：生成6标准视角的高质量产品图，作为全片视觉锚点。

**6标准视角**：
1. 正前方（Front）
2. 左前45°（Front 3/4）
3. 正侧面（Side）
4. 右后45°（Rear 3/4）
5. 正后方（Rear）
6. 俯视（Top）

**生成策略**：
- 分2批生成，每批3张（前3视角 + 后3视角），避免单次6张导致质量稀释
- 使用 `image_to_images` 工具，以用户认可的最佳原图为参考
- Prompt必须包含：设计语言、材质描述、灯光要求、禁止项
- 每个视角单独描述其可见设计元素，不要笼统描述

**Prompt结构模板**：
```
[品质声明：世界级设计标准参照对象]
[产品规格：尺寸/定位/价格带]
[设计语言：按视角描述可见元素]
[材质要求：每种材质的视觉质感]
[6视角逐一描述：每视角一段]
[禁止项：零Logo/零文字/零标注]
[背景与灯光：纯色背景+戏剧性工作室光]
```

**质量验证清单**：
- [ ] 6视角造型逻辑一致（同一台车）
- [ ] 无Logo/无文字/无数字/无标注
- [ ] 材质质感到位（金属漆有深度、镀铬有反射）
- [ ] 设计签名元素每个视角都正确呈现
- [ ] 比例正确（长宽高关系协调）

**迭代规则**：
- 用户说"不够好"→ 重写Prompt重生成，不微调旧图
- 用户说"有进步但还差"→ 针对性补强弱项再生成

## 三、脚本设计流程

### 3.1 时长与镜数规划

**压缩原则**：
- 目标90秒：7镜结构（10s + 6×14s + 10s）
- 每镜覆盖1-2个Pipeline步骤 + 1-3个Claim + 1-2个公式
- 宁可合并相关内容，不设纯过渡镜

**时长分配参考**：

| 镜号 | 时长 | 功能 |
|------|------|------|
| 1 | 10s | 开场：产品登场 + 标题 + Pipeline总览 |
| 2-6 | 各14s | 核心步骤：硬点→包络→融合→特征线→精修→验证 |
| 7 | 10s | 收尾：性能指标 + 致谢 |

### 3.2 Prompt写作规范

**每镜Prompt必须包含**：
1. 承接语：`Continuing from [上一镜状态]`——确保视频生成模型理解上下文
2. 步骤标签：`Step N: [步骤名]`——视觉上标明当前阶段
3. 车身状态描述：当前车的渲染状态（线框/半透明/全渲染）
4. 动画描述：数据可视化动画（曲线绘制、公式卡片、扫描线等）
5. 公式卡片：关键公式的可视化呈现方式
6. 镜头运动：Camera push/pull/orbit/track
7. 背景色：统一 `#F0F4F8`（信息图阶段）/ 深炭灰（开场收尾）
8. 英文副标题：`English subtitle: '...'。不添加字幕，不添加BGM`

**色彩体系**：
- 深蓝 `#2C5282`：硬点、曲线、公式高亮、步骤标签
- 工程灰 `#4A5568`：坐标轴、辅助线、次要标注
- 亮绿：G0验证通过
- 暖琥珀金：光子线（产品签名色）
- 深红：心跳尾灯（产品签名色）

**公式卡片样式**：
- 半透明毛玻璃白底
- 关键变量用深蓝高亮
- 飞入动画 0.2s ease-out

### 3.3 素材绑定

每镜 `materials` 数组绑定策略：
- 至少1个车身参考图（根据该镜主要视角选择）
- 相关PPT参考页
- 跨视角镜头绑定2-3个车身参考

```
示例：

## 四、生成与交付流程

```

## 五、避坑清单

| 踩过的坑 | 解法 |
|----------|------|
| AI生图车身反复出现文字/Logo/标注 | Prompt末尾强指令 + 每次生成后人工验证 + 不合格就重生成不修补 |
| 6视角造型不一致 | 分2批生成、用同一批参考图、每视角单独描述设计元素 |
| 16镜180秒积分浪费 | 压缩到7镜90秒，合并相关步骤，砍纯过渡镜 |
| 参考图质量不够"世界级" | Prompt加入品质声明（对标Maybach/Rolls-Royce发布会）、逐视角描述材质质感、强调"intentionally designed not generic" |
| 公式可视化与车身脱节 | 每个公式卡片出现时，对应车身区域同步高亮/动画 |
| 前后镜视觉不连贯 | 每镜以"Continuing from..."开头，描述当前车身渲染状态 |
| 素材入库参数格式错误 | 道具用 `--item`，角色用 `--role`，替换加 `--replace` |
