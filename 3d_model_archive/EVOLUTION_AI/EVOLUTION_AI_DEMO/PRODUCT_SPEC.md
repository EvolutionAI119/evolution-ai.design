## 1. 业务背景与机会

### 1.1 行业现状

汽车 A 级曲面开发是造型→工程链条上**最耗时、最易出错的环节之一**：

| 痛点 | 现状 | 影响 |
|------|------|------|
| 周期长 | 整车 A 级曲面从概念到 CAS 需 3–4 周 | 上市节奏被卡、研发成本高 |
| 反复修改 | 一次造型冻结后改 30+ 次是常态 | 设计师产能被消磨 |
| 质量不稳定 | F5/F6/F7 检查靠人工经验 | 不达标导致油泥→数字反复 |
| 数据交接易丢信息 | IGES/STEP/JT 转换时控制点丢失 | 工程师拿到数据要重做 |
| 工具碎片化 | Alias / ICEM / CATIA / Rhino 切换成本高 | 团队协作效率低 |

### 1.2 市场机会

- **AI 替代重复劳动**：曲面光顺、参数化联动、连续性检查是高度可自动化的环节
- **造型→工程一体化**：用一个平台打通「参数输入→曲面生成→质量评估→数据交付」全链路
- **行业渗透加速**：截至 2026 年 Q2，TOP 10 主机厂均在评估 AI 辅助造型工具

### 1.3 EVOLUTION AI 的切入点

**做"AI 驱动的 A 级曲面全流程平台"——而不是单点工具**：
- 不抢造型师饭碗，做"光顺优化 + 质量守卫 + 数据交付"三个高价值自动化环节
- 不替代 Alias/CATIA，做"上游参数入口 + 下游交付出口"
- 不做"酷炫 Demo"收尾，做"装进主机厂工具链的真产品"

## 2. 产品愿景与目标

### 2.1 愿景（Vision）

> **让 A 级曲面开发从"经验手艺"变成"AI 工程"——周期减半，质量可控，造型师的精力回归创意本身。**

### 2.2 目标（Goals）

| 维度 | 目标 | 度量 |
|------|------|------|
| 效率 | 整车 A 级曲面开发周期 3–4 周 → 1–1.5 周 | 缩短 60%+ |
| 精度 | 工程交付标准 ±0.1mm | 与手工建模对齐 |
| 质量 | F5/F6/F7 自动检查覆盖率 100% | 不依赖人工抽检 |
| 协同 | 参数化联动响应 < 1s | 实时 |
| 落地 | 1 套工具覆盖"建模→优化→质检→交付"全链 | 替换 3+ 工具碎片 |

### 2.3 非目标（Non-Goals）

- **不做**：纯造型创意工具（Alias/Rhino 形态创作）
- **不做**：CAE 仿真（结构/碰撞/CFD，由下游 HyperWorks/StarCCM+ 负责）
- **不做**：二维效果图渲染（VR/HDRI 由 KeyShot/Octane 负责）
- **不做**：项目管理 PM 工具（用飞书/Jira 即可）
- **不做**：3D 打印/油泥模型接口（线下工艺，独立链路）

## 5. 功能详细规格

### M-01 · 参数化建模引擎

**目的**：用 10+ 个整车级参数驱动完整汽车造型生成。

**输入**：10+ 维参数向量
- 基础尺寸：L（车长）、W（车宽）、H（车高）、wheelbase（轴距）
- 比例姿态：hood_length（机盖长度）、cabin_length（座舱长度）、trunk_length（行李箱长度）、ground_clearance（离地间隙）
- 曲面特征：hood_angle（机盖角度）、roof_arc（车顶弧度）、windshield_rake（前挡风倾角）、rear_glass_angle（后挡风角度）、fender_prominence（轮眉突出度）、waist_line（腰线高度）、shoulder_line（肩线宽度）、overall_arc（整体弧度）

**输出**：完整汽车 trimesh 网格（车壳 + 玻璃 + 4 轮 + 大灯 + 尾灯 + 格栅 + 后视镜 + 门缝线）

**核心算法**：
- 主车身：截面扫掠（24 段车长方向 × 24 段圆周方向）
- 截面形状：超椭圆 + 修形（上窄下宽、肩部最宽）
- 各部件：分别构造再 `trimesh.util.concatenate`

**性能指标**：
- 网格规模：~10K 顶点（车壳）+ ~5K（4 轮+辐条）= ~15K
- 单次生成耗时：< 200ms（CPU）

**接口**：
```python
def build_full_car(params: CarParams) -> Dict[str, trimesh.Trimesh]
def compute_stats(parts: Dict) -> Dict[str, Any]
```

**验收标准**：
- [ ] 默认参数下能生成结构完整、可渲染的汽车造型
- [ ] 所有 10+ 参数改变后，造型肉眼可见地变化
- [ ] 边界值（如 L=5.5 / L=3.5）不发生 mesh 退化

---

### M-02 · 实时编辑器

**目的**：浏览器内 WebGL 渲染 3D 汽车，参数滑块拖动 → 1s 内造型实时更新。

**前端技术栈**：
- Three.js 0.160+（WebGL 渲染）
- 原生 HTML/CSS/JS（轻量、无构建步骤）
- WebSocket 或 Server-Sent Events（参数推送）

**视图能力**：
- 360° 旋转 / 缩放 / 平移
- 4 个预设视角：前 3/4 / 侧视 / 顶视 / 后 3/4
- 部件显隐：车壳 / 玻璃 / 轮 / 灯 / 格栅 / 镜 / 门缝（独立勾选）
- 灯光：3 点光（环境光 + 主光 + 补光）+ HDR 环境贴图（金属感）
- 材质：金属车漆（高光）/ 玻璃（透射）/ 橡胶（哑光）/ 灯罩（自发光）

**交互流程**：
1. 用户拖动滑块
2. 前端 debounce 100ms 后向 `/api/v1/parameters/update` POST
3. 后端重建 mesh（< 200ms）
4. 后端返回 GLB 二进制
5. 前端替换 Three.js scene 中 mesh

**性能指标**：
- 参数联动响应 < 1s（端到端）
- 旋转 60fps
- 移动端可用（响应式布局）

**接口**：
```
POST /api/v1/parameters/update
GET  /api/v1/parameters/default
GET  /api/v1/parameters/schema
GET  /api/v1/model.glb?param=...
```

**验收标准**：
- [ ] 浏览器打开 `http://localhost:8000` 直接看到完整汽车
- [ ] 拖动任一滑块，1s 内看到车身变形
- [ ] 4 个预设视角按钮可一键切换
- [ ] 部件可独立显隐

### M-03 · AI 优化引擎

**目的**：把 B 级曲面自动优化到 A 级，解决"反复调控制点"痛点。

**目标函数**：
- 主目标：法向量跳变平方和（光顺度度量）
- 约束项：边界控制点不变（保持与相邻曲面拼接）
- 惩罚项：超出物理合理范围的扰动

**算法选型（MVP 用模拟退火）**：
| 算法 | 优点 | 缺点 | 适用阶段 |
|------|------|------|----------|
| 模拟退火 | 简单稳定、易实现 | 收敛慢 | MVP ✅ |
| 遗传算法 | 全局搜索强 | 参数多 | 后续优化 |
| 贝叶斯优化 | 样本高效 | 高维慢 | 后续优化 |
| 神经代理模型 | 极快 | 需训练数据 | 长期 |

**API 形态**：
```python
def ai_optimize_surface(panel_name, params, max_iter=150) -> OptimizationResult
```

**输出**：
- 优化后的控制点
- 收敛曲线（用于可视化）
- 改进幅度（G2 数 / 反射线评分 / 等级）

**性能指标**：
- 150 次迭代 < 30s
- 改进幅度 ≥ 20%（光顺度评分）
- 至少 1 个面板能从 B 升到 A

**验收标准**：
- [ ] 默认参数下，侧围从 B 升 A
- [ ] G2 连续点数提升 ≥ 20%
- [ ] 反射线评分提升 ≥ 0.08
- [ ] 收敛曲线可绘制

---

### M-04 · 质量评估器

**目的**：自动计算 G0/G1/G2 连续性 + 反射线评分 + 综合等级。

**指标体系**：

| 指标 | 计算方法 | A 级要求 | 工程意义 |
|------|----------|----------|----------|
| G0 | 位置连续 | 100% | 曲面无缝隙 |
| G1 | 法向夹角 < 5° | > 85% | 一阶光滑 |
| G2 | 法向夹角 < 2° | > 85% | 二阶光滑（反射线无折） |
| 反射线评分 | 曲率均匀度 + 平滑度 | > 0.7 | 视觉高级感 |
| 最大曲率跳变 | max(相邻点曲率差) | < 1° | 局部无瑕疵 |
| 综合等级 | A/B/C/D | - | 造型冻结门槛 |

**API 形态**：
```python
def assess_quality(panel_name, params) -> QualityReport
```

**输出**：
- JSON 报告（含所有指标）
- HTML 可视化报告（反射线 + 曲率云图 + 数据表）
- 等级徽章（A/B/C/D）

**验收标准**：
- [ ] 所有指标均能计算并输出
- [ ] A/B/C/D 等级判定与汽车行业经验一致
- [ ] HTML 报告可直接打开（无需服务器）

## 10. 名词术语表

| 术语 | 英文 | 定义 |

| A 级曲面 | Class A Surface | 汽车行业最高质量等级的曲面，视觉无瑕、反射线连续 |
| CAS | Computer Aided Styling | 计算机辅助造型设计 |
| Bezier 曲面 | Bezier Surface | 基于控制点的参数化曲面，工业造型最常用 |
| NURBS | Non-Uniform Rational B-Spline | 非均匀有理 B 样条，CAD 行业标准 |
| G0 连续 | G0 Continuity | 位置连续（曲面无缝隙） |
| G1 连续 | G1 Continuity | 一阶导数连续（切线方向连续） |
| G2 连续 | G2 Continuity | 二阶导数连续（曲率连续，反射线无折） |
| F5/F6/F7 | F5/F6/F7 Check | 行业标准曲面质量检查等级 |
| 反射线 | Reflection Line | 模拟环境光在曲面上的反射轨迹，肉眼判断 A 级曲面关键 |

| 模拟退火 | Simulated Annealing | 受金属退火启发的全局优化算法 |
| 腰线 | Waist Line | 车身侧面中部横向特征线 |
| 肩线 | Shoulder Line | 车身侧面靠近车顶的横向特征线 |
| 轮眉 | Fender | 车轮上方的车身边缘 |
| 离地间隙 | Ground Clearance | 车身最低点距地面距离 |
