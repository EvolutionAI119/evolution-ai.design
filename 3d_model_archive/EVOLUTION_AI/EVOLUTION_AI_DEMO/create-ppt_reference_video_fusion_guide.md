## 三、Shot → PPT 页面映射策略

### 3.1 映射原则

1. **语义匹配**：shot 的 `semantic_desc` 与 PPT 页面主题内容一致或高度相关
2. **叙事顺承**：shot 在视频中的顺序应与 PPT 页面的叙事逻辑大体一致
3. **封面优先**：视频开篇 shot（通常是最具视觉冲击力的画面）优先分配给 PPT 封面
4. **结尾收束**：视频结尾 shot（Logo/品牌画面）分配给 PPT 结尾页
5. **对比成对**：同一主题的对比类 shot（如优化前/后）应分配到同一页面
6. **每页限图**：每个 PPT 页面最多使用 2 张首帧图作为 ref_images
7. **不重复**：每张首帧图只能在一个 PPT 页面使用

### 3.2 标准映射流程

```
Step 1: 遍历所有 shots，提取 semantic_desc 并标注主题标签
Step 2: 规划 PPT 页面结构（封面 + 内容页 + 结尾）
Step 3: 按以下规则分配 shot：
  - shot #1（开篇） → 封面页
  - shot #N（结尾/Logo） → 结尾页
  - 主题匹配的 shot → 对应内容页
  - 无明确匹配的 shot → 选择内容最相关的页面
Step 4: 检查是否所有 shot 都已分配，或合理弃用部分 shot
Step 5: 输出映射表供确认
```

### 3.3 常见映射模式

| PPT 页面类型 | 推荐映射的 shot 类型 |
|-------------|-------------------|
| 封面页 | 视频开篇镜头（视觉冲击力最强） |
| 痛点/背景页 | 通常不配首帧图（纯信息图） |
| 产品概述页 | 产品/车型全貌展示 shot |

## 四、Prompt 编写规范

### 4.1 布局预留

当某页 PPT 使用首帧图作为 ref_images 时，必须在画面描述中**显式预留图片区域**：

```
页面左侧占45%放置概念车展示图，右侧占55%展示三大核心定位：
```

**禁止**只在参考图说明中写位置而不在画面描述中预留空间。

### 4.2 参考图说明写法

融合模式下的首帧图统一使用「配图嵌入」模式（Logo 类除外）：

**单图（产品/展示图）**：
```
参考图说明（以下内容仅用于指导参考图的使用方式，不要把文字本身写进画面）：
提供的参考图是{图片描述}，放置在{位置}。照片主体内容必须保持不变，可以添加与页面风格协调的边框装饰。
```

**双图（对比/多角度）**：
```
参考图说明（以下内容仅用于指导参考图的使用方式，不要把文字本身写进画面）：
第一张参考图是{图片描述}，放置在{位置1}。照片主体内容必须保持不变，可以添加与页面风格协调的边框装饰。
第二张参考图是{图片描述}，放置在{位置2}。照片主体内容必须保持不变，可以添加与页面风格协调的边框装饰。
```

**Logo/品牌标识**（严格原样）：
```
参考图说明（以下内容仅用于指导参考图的使用方式，不要把文字本身写进画面）：
提供的参考图是{品牌Logo描述}，放置在{位置}。必须作为独立图层原样嵌入，不添加任何边框或装饰，保持原始比例不拉伸。
```

### 4.3 内容与图意互补

融合模式的核心价值在于**图文互补**而非图文重复：

- 首帧图展示的是**视觉结果**（车型、对比效果、参数界面）
- PPT 文字应补充图**未说清的**信息（数据指标、操作步骤、业务价值）
- 避免文字重复描述图中已清晰呈现的内容

## 五、中英文双语生成

当用户要求同时生成中英文版本时：

1. 先生成中文版，确认所有页面成功
2. 使用相同的映射方案和首帧图，将内容翻译为英文后生成英文版
3. 两版 PPT 的 `ppt_title` 应区分（如添加后缀 `_Fusion` / `_融合版`）
4. 两版共享相同的首帧图 URL 和映射逻辑
5. prompt 中的参考图说明需与页面内容语言一致

## 六、注意事项

1. **首帧图质量**：AIGC 生成的首帧图可能存在文字模糊问题，这是模型能力限制。如果用户反馈文字不清晰，建议走「去文字重生成」或「后期叠加文字」路径
2. **资产保护**：视频脚本中已成功生成的 shot 视频受资产保护规则约束，修改需用户明确确认范围
3. **shot 数据完整性**：如果 shots_all.json 不存在或不完整，应暂停并告知用户缺少视频脚本数据
4. **图片 URL 有效性**：首帧图 URL 有时效性，如果生成 PPT 时 URL 失效，需重新入库获取
