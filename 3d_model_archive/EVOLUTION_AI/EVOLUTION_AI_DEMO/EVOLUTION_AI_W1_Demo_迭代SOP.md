## 四、制作流程

### 阶段 1：脚本初始化
1. 调用 `init_video_script` 初始化脚本文件，设置标题、画幅比例
2. 上传/确认所有参考图片素材（3D 模型、UI 界面、Logo）
3. 系统自动将素材写入素材库

### 阶段 2：分镜撰写
1. 按叙事线拆分 7 个 shot
2. 每个 shot 填写：prompt（画面描述）、duration（时长）、镜头语言、特效描述
3. 参考素材通过素材名称引用（如 `3D_Car_Model`）
4. 多轮打磨：检查前后镜衔接、特效不重复、叙事节奏

### 阶段 3：状态检查与消耗预估
1. 执行视频状态检查，确认所有 shot 字段完整
2. 执行消耗预估，向用户展示总积分消耗
3. 用户确认后进入生成阶段

### 阶段 4：批量生成
1. 调用批量生成工具，一次生成全部 7 镜

### 阶段 5：工程组装

## 五、迭代指南

### 场景 A：替换车型
1. 准备新车型参考图（3/4 侧面角度最佳）
2. 替换素材库中的 `3D_Car_Model`
3. 检查每个引用该素材的 shot，确认 prompt 中车型描述与新图一致
4. 重新生成全部 7 镜

### 场景 B：调整叙事节奏
1. 修改对应 shot 的 `duration` 字段
2. 若新增/删除 shot，需重新检查前后镜衔接关系
3. 更新分镜结构表

### 场景 C：升级视觉特效
1. 定位目标 shot 的 prompt 中的特效描述段落
2. 替换为新的特效描述，保持与整体风格一致
3. 单独重新生成该 shot 验证效果

### 场景 D：加入旁白/字幕/BGM
1. 在脚本中为每个 shot 添加旁白文本字段
2. 调用 TTS 工具生成旁白音频
3. 调用选曲工具或自定义生成 BGM
4. 重新执行工程组装

### 场景 E：调整 UI 界面
1. 重新生成或替换对应 UI 素材图
2. 更新素材库引用
3. 检查相关 shot 的 prompt 是否需要同步修改界面描述
4. 重新生成受影响的 shot

## 六、关键经验总结

1. **素材先行**：所有参考图在写分镜前就位，能大幅减少 prompt 中的模糊描述
2. **一镜一特效**：避免全片特效堆砌，每镜一个记忆点比每镜三个特效更有效
3. **前后镜衔接要显式标注**：在 prompt 中明确写"接住前一镜的 XX 状态"，生成时系统能更好保持连续性
4. **Shot 职责分离是打磨重点**：相邻叙事阶段的 shot 容易功能重叠，需在打磨阶段明确各自重心
5. **消耗预估必做**：7 镜 90s 约消耗 9 万积分，提前确认避免中断
