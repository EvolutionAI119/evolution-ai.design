#### Prompt 编写要点

融合模式下的 prompt 与纯内容模式有三个关键差异：

1. **布局预留**：画面描述中必须显式标注图片区域位置与占比
   ```

   ```

2. **图文互补**：文字补充图中未说清的信息，避免重复描述图中已有内容

3. **参考图说明**：首帧图使用「配图嵌入」模式，Logo 使用「严格原样」模式

## 六、经验与反思

### 6.1 关键发现

1. **AIGC 视频文字是硬限制**：当前视频生成模型无法清晰渲染小字号文字，这属于 prompt_policy 类问题，不是 prompt 优化能解决的。建议长期方案是「去文字生成纯净画面 + 后期叠加」。

2. **视频首帧图是跨技能复用的最佳资产**：首帧图既是视频的视觉起点，又是 PPT 的高质量配图。一次生成、两处复用，显著提升物料一致性。

3. **Shot 语义描述是映射的关键输入**：`semantic_desc` 比完整 prompt 更适合做映射决策——它凝练了画面主题，而 prompt 包含大量渲染细节。

### 6.2 流程改进建议

1. **映射表标准化**：未来可考虑将映射表结构化为 JSON，便于自动化处理和跨项目复用
2. **首帧图 URL 自动提取**：当前需要手动从素材库或生成结果中提取 URL，可编写脚本自动从 shots_all.json + 素材库中聚合
3. **双语并行生成**：中英文版本共享映射方案，可考虑一次 JSON 调用同时生成两版，减少重复工作
4. **文字清晰度分级**：可建立「必须清晰文字」与「可接受模糊文字」的分级标注，在 shot 级别决定是否去文字重生成
